# Security Policy

## Scope

This repository is a research demo app. It runs LLM-driven social simulations
through a Streamlit UI and calls out to user-supplied LLM provider APIs
(Gemini by default; any OpenAI-compatible endpoint as the second track).

The security model prioritises:

1. **BYOK (Bring Your Own Key).** No API key is ever stored, logged, or
   transmitted server-side. Keys live only in a user's in-memory Streamlit
   `session_state` and are forgotten on tab close or after 30 minutes of
   inactivity.
2. **Supply-chain containment.** `litellm` is intentionally not used; the
   project depends on `google-genai` and `httpx` only for LLM transport.
   All dependencies are version-pinned in `requirements.txt` and
   hash-pinned in `requirements-lock.txt`.
3. **Prompt-injection resistance.** User-supplied text (agent profiles,
   topics, intervention messages) is wrapped with spotlighting delimiters
   and scanned for role-takeover, invisible-character, and encoding-bypass
   patterns before being sent to the model.
4. **Output isolation.** Model responses are filtered before Streamlit
   rendering: HTML stripped, non-HTTPS links dropped, inline images
   removed. `unsafe_allow_html` is never enabled.
5. **Consumption bounds.** Each session has a token budget (500 K) and an
   approximate cost ceiling ($2) in addition to per-session rate limits.

## Threat model

| Threat | Mitigation |
|---|---|
| OWASP LLM01 Prompt Injection | [app/security.py](app/security.py) `spotlight()`, `sanitize_user_input()`, invisible-char stripping |
| OWASP LLM02 Sensitive Information Disclosure | System prompt is minimal, `sanitize_llm_output()` strips echoed delimiters |
| OWASP LLM03 Supply Chain | Pinned deps + `requirements-lock.txt` (hash-pinned), no `litellm`, no `openai` SDK |
| OWASP LLM05 Improper Output Handling | `sanitize_llm_output()` before every `st.markdown` / `st.write` |
| OWASP LLM07 System Prompt Leakage | Delimiter echo removed from output; system prompt holds no secrets |
| OWASP LLM10 Unbounded Consumption | `account_tokens()` session ceiling + `cap()` parameter caps |
| Path Traversal (CWE-22) | `safe_profile_path()` — allowlist regex + `is_relative_to` |
| Secret leakage via logs / errors | `sanitize_error()` redacts Gemini / OpenAI / Anthropic / GitHub key patterns |
| Streamlit XSS / CSRF | `unsafe_allow_html=False` everywhere, XSRF protection on in `.streamlit/config.toml` |

## Reporting a vulnerability

Please do **not** open a public GitHub issue. Email the maintainer at
`jysuh@sapiensq.com` with:

- a description of the issue,
- steps to reproduce,
- affected version / commit hash,
- your name / handle for credit (optional).

Expect an acknowledgement within 3 business days. We will coordinate a fix
and disclosure timeline with you before publishing.

## Supported versions

Only the latest release on the `main` branch is supported. Paper-specific
(`paper-*`) and examples-specific (`examples-*`) feature branches receive
security merges from `main` but are not independently supported.

## Dependency scanning

- `requirements.txt` pins every direct dependency with a bounded semver
  range (`>=X.Y.Z,<N+1.0.0`).
- `requirements-lock.txt` pins every transitive dependency with SHA-256
  hashes, generated by `pip-compile --generate-hashes`.
- For reproducible installs, use:
  ```bash
  pip install --require-hashes -r requirements-lock.txt
  ```
- Recommended CI gates (not yet enforced in this repo):
  - `pip-audit --strict`
  - `safety check`
  - `osv-scanner --recursive .`

## Key rotation guidance

If you suspect your BYOK API key has been exposed (e.g., pasted into a
shared notebook, captured in logs), revoke it immediately at:

- Gemini: https://aistudio.google.com/apikey
- OpenAI: https://platform.openai.com/api-keys
- Anthropic: https://console.anthropic.com/settings/keys

The app never persists keys, but other systems you copy-pasted into may.
